## üöÄ Publishing Workflow

### For Each App:

1. **Build Signed AAB:**
   ```bash
   ./build_signed_release.sh <developer> <app_name>
   ```

2. **Generate Store Assets:**
   ```bash
   ./generate_store_assets.sh <developer> <app_name>
   ```

3. **Publish to GPC:**
   ```bash
   ./publish_to_gpc.sh <developer> <app_name> <track>
   ```

### Example - Publish joke_generator:

```bash
# 1. Build
./build_signed_release.sh 01_giggle_game joke_generator

# 2. Generate assets
./generate_store_assets.sh 01_giggle_game joke_generator

# 3. Publish
./publish_to_gpc.sh 01_giggle_game joke_generator production
```

Or use the master workflow (all-in-one):
```bash
./publish_to_gpc.sh 01_giggle_game joke_generator production
```

## üìã GPC Account Setup Checklist

For each GPC account, you need:

- [ ] Service account JSON (for API access)
- [ ] API access enabled in Google Play Console
- [ ] Service account linked in GPC ‚Üí Setup ‚Üí API access
- [ ] GitHub Secret: `GPC_SERVICE_ACCOUNT_JSON_{ACCOUNT}` (base64 encoded)
- [ ] GitHub Secret: `GPC_PACKAGE_NAME_{APP}` (optional, per app)
